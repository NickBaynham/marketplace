# Anomaly Classification Checklist

Apply to each anomaly row in the anomaly report. Each item should be Pass, Fail, or Needs Review.

| Check | Status | Notes |
|---|---|---|
| Anomaly type identified (Application, Tooling, Environment, Data, Usability, Accessibility, Performance, Unknown) | | |
| Severity assigned (Low, Medium, High, Critical, Needs Clarification) | | |
| Evidence recorded (snapshot, screenshot, console excerpt, network request, URL) | | |
| Application behavior separated from tooling behavior | | |
| Environment issues separated from application issues | | |
| Recommendation provided | | |
| Status assigned (Open, Needs Review, Clarified, Potential Defect, Tooling Limitation, Closed) | | |
| Potential defects marked clearly as Potential Defect, not asserted as Defect | | |
| Uncertainty documented (what is unknown, what would clarify it) | | |
